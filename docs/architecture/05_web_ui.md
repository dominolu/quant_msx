# FastAPI Web 控制台

## 1. 首屏

默认进入网格策略工作台，不做营销页。页面由 FastAPI 直接托管，不使用独立 React/Vite 前端工程。

首屏区域：

- 顶部系统状态条：MSX 连接、交易开关、运行策略数、异常数。
- 左侧策略列表或导航。
- 主区域展示网格列表和摘要。

## 2. 网格列表

字段：

- 名称、市场、交易对、状态。
- 方向、杠杆、保证金模式。
- 网格间距、单笔数量、价格区间。
- 当前价格、基准价、上下挂单价。
- 当前持仓、今日成交、累计成交。
- 网格收益、总 PnL、健康状态。
- 最近更新时间。

操作：

- 创建。
- 启动。
- 暂停。
- 恢复。
- 停止。
- 重配。
- 查看详情。

## 3. 网格详情

区域：

- 参数摘要。
- 当前挂单。
- 成交记录。
- 收益曲线。
- 事件日志。
- 风控状态。

## 4. 系统页面

- API Key 配置状态，不展示密钥明文。
- MSX REST / WS 连接状态。
- 限流、错误、同步统计。
- 全局 kill switch。

## 5. 交互原则

- 实盘危险操作必须二次确认。
- 暂停、停止、重配必须展示即将撤销的订单数。
- 平仓动作必须独立确认，不和停止默认绑定。
- 错误提示直接展示后台拒绝原因。

## 6. 实现方式

首期采用：

- `app/web/routes.py` 返回 HTML 页面。
- `app/web/static/` 存放 CSS 和轻量 JavaScript。
- 页面通过 `/api/*` 读取 JSON 数据。

后续如页面复杂度上升，优先考虑 HTMX 或 Alpine.js 这类无需构建链的增强方案。只有在交互复杂度明显超过 FastAPI 托管页面能力时，才重新评估独立前端工程。
