# 交易执行与风控

## 1. 交易开关

默认 `LIVE_TRADING_ENABLED=false`。任何真实下单、撤单、平仓动作必须满足：

- 环境变量显式开启。
- API Key 已配置。
- 策略状态允许交易。
- 全局 kill switch 未触发。

## 2. 下单前检查

- symbol 存在。
- 市场类型正确。
- 价格满足 tick size。
- 数量满足 lot size。
- 名义金额满足最小下单额。
- 网格价格在策略边界内。
- 当前净持仓不超过 `max_position_qty`。
- 单策略挂单数不超过限制。
- 账户余额或保证金充足。
- 下单频率不超过 MSX 限流约束。

## 3. 运行中检查

- WebSocket 心跳超时。
- REST 查询失败率。
- 订单长时间未回报。
- 本地挂单与交易所当前委托不一致。
- 当前亏损超过 `max_loss_usdt`。
- 当前价格触发止盈或止损。

## 4. 停止策略

停止动作分两层：

- `pause`: 撤销未成交订单，不主动平仓，保留策略。
- `stop`: 撤销未成交订单，并按策略配置决定是否平掉关联持仓。

首期默认停止时只撤单，平仓必须后续明确设计并在 UI 上显式确认。

## 5. 对账策略

触发条件：

- 服务启动。
- WebSocket 重连成功。
- 订单状态超过超时时间未更新。
- 用户手动点击同步。

同步内容：

- 当前委托。
- 历史订单。
- 成交记录。
- 合约持仓或现货余额。

同步结果必须写入 `GridEvent`。
