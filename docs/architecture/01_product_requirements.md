# 产品需求

## 1. 背景

`quant_msx` 是一个面向 MSX 平台的量化交易系统。首期目标不是做通用交易框架，而是围绕 MSX 现货与合约 API，快速形成可运行、可监控、可恢复的网格策略平台。

参考项目 `/Users/dominolu/dev/gate_arb` 的后端分层方式，但本项目不采用独立前端工程。Web 控制台由 FastAPI 直接托管，降低部署复杂度；产品重心从跨交易所套利切换为 MSX 网格交易。

## 2. 目标用户

- 运行 MSX 现货或合约网格的个人量化交易者。
- 需要 Web 控制台管理多组网格策略的交易运维人员。
- 需要清晰审计策略状态、订单状态和异常恢复过程的开发者。

## 3. 首期目标

- 接入 MSX 现货和合约 REST API。
- 接入 MSX 现货和合约 WebSocket 行情。
- 支持创建、启动、暂停、恢复、停止网格策略。
- 支持网格运行中的订单、成交、收益、事件展示。
- 支持服务重启后的策略恢复和 REST 补同步。
- 支持最小可用的预交易风控和运行中 kill switch。

## 4. 非目标

- 不做历史回测系统。
- 不做自动选币和自动调参。
- 不做多交易所套利。
- 不做高频盘口做市。
- 不做资金托管或权限管理系统。

## 5. 核心功能

### 5.1 MSX 接入

- API Key 签名和请求头封装。
- REST 统一请求和错误归一化。
- WebSocket 自动重连、心跳、订阅恢复。
- 现货与合约接口隔离，避免路径和字段混用。

### 5.2 网格策略

- 每个网格实例绑定一个 market + symbol。
- 支持 `spot` 和 `futures` 两种市场。
- 支持 `neutral`、`long_bias`、`short_bias` 方向。
- 支持百分比网格和固定价差网格。
- 每轮维持上下两侧挂单，成交后撤销对侧并重置基准价。
- 支持价格区间、最大仓位、止盈止损、最大亏损。

### 5.3 风控

- 下单前校验交易开关、账户、市场、精度、最小下单量。
- 校验最大持仓、最大挂单数、最大亏损、价格边界。
- 检测 WebSocket 失联、REST 补同步失败、订单状态不一致。
- 支持全局 kill switch 和单策略停止。

### 5.4 控制台

- 网格列表与摘要。
- 网格详情、当前挂单、成交记录、事件日志。
- 手动启动、暂停、恢复、停止、重配。
- 行情、连接状态、REST 补同步次数、错误提示。

## 6. 成功标准

- 本地可启动后端并通过健康检查。
- 可使用 MSX API 文档实现签名和公开行情查询。
- 网格策略可以在 demo 模式下完整走通状态机。
- 打开真实交易开关前，所有实盘入口都有显式配置保护。
- 后续扩展 FastAPI Web 页面时无需重写核心领域模型。
